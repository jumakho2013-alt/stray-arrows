# Аудит проекта Stray Arrows, 2026-04-28

## TL;DR

Игра **технически готова к релизу** на 90%. iOS v1.0.7 уже live в App Store, Android v1.0.9 готов в коде к сборке. Код чистый, без явных костылей, с хорошим i18n (9 языков, паритет), правильной интеграцией AdMob и безопасной работой с localStorage.

**Главные риски:**
1. 🔴 **Пароль keystore в открытом виде в `android/app/build.gradle`** — пушнуто в публичный репо. Любой кто склонирует репо видит пароль.
2. 🔴 **Бэкап keystore не подтверждён** — если файл потеряется, Android-приложение **никогда** не сможет обновиться (Google Play требует ту же подпись).
3. 🟡 AdMob listeners накапливаются при ре-инициализации (после backgrounding на iOS) — теоретически приводит к двойным наградам.
4. 🟡 `localStorage.setItem` не защищён от исключения в Safari Private Browsing — приложение упадёт при попытке сохранить прогресс.
5. 🟡 Android `allowBackup="true"` — данные бэкапятся в облако без шифрования.

**Что делать в первую очередь:** ротировать Android keystore (не из-за компрометации, а потому что пароль в git history) и зашифровать бэкап. Остальное — желательно, но не блокирует релиз.

## Критические проблемы (must fix)

### 1. Пароль keystore в `android/app/build.gradle`
```gradle
storePassword 'ArrowEscape2024'
keyPassword 'ArrowEscape2024'
```
- **Файлы**: `android/app/build.gradle:22,24`
- **Суть**: Пароли в plaintext, в публичной истории git (commits начиная с раннего Codex-PR)
- **Кто использует**: любой кто видит репозиторий на GitHub
- **Риск**: если кто-то заполучит ALSO файл keystore (через утечку, потерянный диск, accidental upload) — может подписать вредоносное обновление и пушнуть в Play Store от твоего имени
- **Корректный фикс**: 
  - Создать `android/keystore.properties` (gitignore)
  - Содержит: `storePassword=...`, `keyPassword=...`, `keyAlias=...`, `storeFile=...`
  - В `build.gradle` читать через стандартный паттерн:
    ```gradle
    def keystorePropertiesFile = rootProject.file("keystore.properties")
    def keystoreProperties = new Properties()
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
    
    signingConfigs {
        release {
            storeFile file(keystoreProperties['storeFile'])
            storePassword keystoreProperties['storePassword']
            keyAlias keystoreProperties['keyAlias']
            keyPassword keystoreProperties['keyPassword']
        }
    }
    ```
  - Добавить `keystore.properties` в `.gitignore`
  - Удалить пароли из текущей версии (старые коммиты остаются, но новые контрибьютеры не увидят)
- **После фикса опционально**: ротировать keystore, обновить Google Play Signing

### 2. Backup keystore не проверен
- **Суть**: Если файл `arrow-escape.keystore` потерян (диск умер, репо удалён, итд) — **навсегда теряешь возможность обновлять Android-приложение**
- **Корректный фикс**:
  1. Скопировать `arrow-escape.keystore` в **минимум 2 безопасных места**:
     - 1Password / Bitwarden (как зашифрованный attachment)
     - Зашифрованный архив на Google Drive / iCloud
  2. Сохранить пароль в надёжном password manager
  3. **Никогда не хранить только на одном диске**

## Важные (should fix)

### 3. localStorage.setItem без guard на Safari Private Browsing
- **Файлы**: `index.html:14,46,54,426-438`
- **Суть**: Safari Private Mode бросает `QuotaExceededError` при `setItem`. У нас try-catch только на `_readSafeArea`, но не на `save()`. Игра упадёт при попытке сохранить прогресс в Private Browsing.
- **Корректный фикс**: обернуть все `localStorage.setItem` через единый helper:
  ```js
  function _safeSet(key, val){
    try{localStorage.setItem(key,val);return true}
    catch(e){return false}
  }
  ```

### 4. AdMob listeners накапливаются на re-init
- **Файлы**: `index.html:606-617`
- **Суть**: Каждый `initAdMob()` вызывает `AdMob.addListener(...)` × 3, без `removeListener`. На background→foreground iOS может пере-инициализировать → 2× listeners → следующий reward вызывает callback дважды.
- **Корректный фикс**: ставить листенеры **один раз** при первом init, или вызывать `removeAllListeners()` перед регистрацией.

### 5. Android `allowBackup="true"`
- **Файлы**: `android/app/src/main/AndroidManifest.xml:4`
- **Суть**: Игровой прогресс (монеты, уровни) бэкапится в Google Drive без шифрования. На скомпрометированном Google аккаунте — viewable.
- **Корректный фикс**: 
  - Либо `allowBackup="false"` (юзеры теряют прогресс при смене устройства)
  - Либо настроить `android:fullBackupContent="@xml/backup_rules"` с явным списком что бэкапить

### 6. iOS NSUserTrackingUsageDescription — расплывчатая формулировка
- **Файлы**: `ios/App/App/Info.plist:51`
- **Текущий текст**: "Your data will be used to deliver personalized ads to you."
- **Риск**: Apple App Review может попросить переписать конкретнее. Не блокирует, но раздражает.
- **Корректный фикс**: 
  ```
  "Allow tracking to deliver more relevant ads. Without permission, you'll still see ads, but they won't be personalized."
  ```

### 7. iOS build number (21) ≠ Android versionCode (16)
- **Файлы**: `ios/App/App.xcodeproj/project.pbxproj` vs `android/app/build.gradle`
- **Суть**: Внутренние номера платформ разъехались. В крэш-репортах будет путаница "1.0.9 build 21" vs "1.0.9 versionCode 16".
- **Корректный фикс**: договориться о синхронизации (например, оба использовать одну цифру) или явно документировать в README что отслеживается отдельно.

### 8. drawPlay() — 224 строки, tryBuildArrow() — 106 строк
- **Файлы**: `index.html:1547-1770`, `index.html:858-963`
- **Суть**: Превышают 100-line guideline. Смешивают рендеринг, input, animations, UI. Сложно тестировать.
- **Корректный фикс**: разбить на:
  - `drawPlayGame()` — рендеринг arrows + grid
  - `drawPlayUI()` — гейр, hint/skip кнопки, top bar
  - `updatePlayAnimations()` — обновление anims/floaters/shakeAnims
  - Для `tryBuildArrow`: вынести scoring formula в отдельную функцию с named constants

### 9. Android `minifyEnabled false`
- **Файлы**: `android/app/build.gradle:30`
- **Суть**: Без R8/ProGuard код не обфусцирован — легче reverse engineering, APK на 30-40% больше.
- **Корректный фикс**: включить `minifyEnabled true` + протестировать (Capacitor + AdMob могут потребовать proguard rules).

### 10. Empty try-catch блоки
- **Файлы**: `index.html:18, 51, 54, 115, 141, 578, 633, 638, 721`
- **Суть**: 5 из них justified (Capacitor fallback). Остальные глотают ошибки беззвучно — невозможно отладить.
- **Корректный фикс**: даже минимальное логирование в `localStorage.ae_silent_err` или объединить в helper `_swallow(fn, label)`.

## Желательные (nice to fix)

### 11. Google Fonts блокирует первый рендер
- **Файлы**: `index.html:27`
- **Суть**: `<link rel="stylesheet">` с `display=swap` — не блокирует фатально, но добавляет ~100-300ms на медленной сети.
- **Корректный фикс**: использовать `display=fallback` + системные шрифты как primary fallback (уже есть `-apple-system, BlinkMacSystemFont`).

### 12. Path caching для повторяющихся форм
- **Файлы**: `index.html` (drawArrowStatic, drawStar, button bg)
- **Суть**: ~2400 path constructions/sec при 60fps × 5 arrows × 8 cells. Не критично на современных iPhone, может стутерить на iPad gen 5.
- **Корректный фикс**: использовать `Path2D` объекты для статичных стрелок и звёзд, кэшировать в Map.

### 13. localStorage.getItem на каждом state transition
- **Файлы**: `index.html:371-419`
- **Суть**: Не в render loop, но при каждом `goto()` пере-читаем 8 ключей. Кешировать в module-level переменные.

### 14. Дубликаты Capacitor type-check
- **Файлы**: `index.html` (8+ мест)
- **Суть**: `typeof Capacitor!=='undefined'` повторяется. Уже есть `_isIOS()`, можно добавить `_hasCapacitor()`.

### 15. Дубликаты button/modal rendering
- **Файлы**: `drawBtn`, `drawBigBtn`, `_drawActionBtn`, drawLangPicker, drawAchPopup
- **Суть**: Каждый сам рисует rounded-rect + border + label. Можно вынести `_drawCard(x,y,w,h,r,bgColor)`.

### 16. inputLocked таймаут — magic number
- **Файлы**: `index.html:1060`
- **Суть**: `setTimeout(...500)` для разлока input после complete. Если анимация изменится, разъедется.
- **Корректный фикс**: использовать `INPUT_LOCK_DURATION` constant, синхронизировать с длительностью fade.

### 17. Crash reporting отсутствует
- **Суть**: Только App Store Connect crash dashboard. Нет stack traces, нет custom событий.
- **Корректный фикс**: добавить Sentry (free tier 5K errors/month) — 10 строк интеграции.
- **Severity**: для solo indie на стадии запуска приемлемо, но через месяц после релиза стоит добавить.

### 18. Нет README / build docs
- **Корректный фикс**: создать README.md с разделами:
  - Quick start (npm install, npm run build, npx cap sync)
  - iOS build flow (Xcode signing, archive, export)
  - Android build flow (keystore, gradle bundleRelease)
  - Where to find: ASC API key (.p8), keystore, signing certs
